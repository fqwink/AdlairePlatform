## §1 本補遺の目的

本補遺は、`FRAMEWORK_RULEBOOK.md`（本則）を補完し、**Ver.2.0 メジャーアップデート**に固有の方針決定・保留事項・不採用判断を記録するものです。

本則に定める規則（エンジン駆動モデル、命名規則、削除禁止事項等）は引き続き有効であり、本補遺はそれらを上書きしません。

---

## §2 Ver.2.0 破壊的変更方針

Ver.2.0 はメジャーバージョンアップであり、**後方互換性のない変更**を含みます。

| 方針 | 内容 | ステータス |
|------|------|-----------|
| バージョニング | Ver.2.0-40（累積リビジョン・リセット禁止 — VERSIONING.md 準拠） | ✅ 確定 |
| 後方互換性 | 破壊的変更を許容する | ✅ 確定 |
| PHP 要件 | PHP 8.3+ 必須（変更なし） | ✅ 確定 |
| フレームワーク構成 | 7 フレームワーク・18 エンジン体制を維持 | ✅ 確定 |

## §3 Ver.2.0 で不採用とした機能

以下の機能は Ver.2.0 での実装を**見送り**ました。

### §3.1 OpenAPI 自動生成

| 項目 | 内容 |
|------|------|
| 機能 | Swagger/OpenAPI 形式の API ドキュメント自動生成 |
| 初出 | Ver.1.3-29 リリースノート「次期バージョンで検討予定」 |

| 理由 | Ver.2.0 スコープ外。API 層の安定化を優先する |
| ステータス | ❌ 不採用（Ver.2.0） |

## §4 保留事項（将来バージョンで再検討）

以下の機能は Ver.1.3-29 リリースノートで「次期バージョンで検討予定」とされたものですが、Ver.2.0 では着手せず**将来バージョンで再検討**とします。

### §4.1 マルチ環境（dev / staging / production）

| 項目 | 内容 |
|------|------|
| 機能 | Git ブランチと環境（開発・ステージング・本番）の連動 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | AIS.Deployment との統合設計が必要。単独で導入するとインフラ複雑性が増大する |

### §4.2 GitHub OAuth

| 項目 | 内容 |
|------|------|
| 機能 | GitHub OAuth による外部認証連携 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | 外部依存ゼロ原則（FRAMEWORK_RULEBOOK §2.2）との整合性を要検討。OAuth ライブラリの自前実装 or 例外許可が必要 |

### §4.3 GraphQL 対応

| 項目 | 内容 |
|------|------|
| 機能 | REST API に加えて GraphQL エンドポイントを提供 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | REST API（ACE.Api）との併存が前提。GraphQL パーサーの自前実装コストが高い |

### §4.4 保留事項サマリー

| 機能 | 初出 | Ver.2.0 判断 | 再検討時期 |
|------|------|-------------|-----------|
| OpenAPI 自動生成 | Ver.1.3-29 | ❌ 不採用 | — |
| マルチ環境 (dev/staging/prod) | Ver.1.3-29 | 🔜 保留 | 未定 |
| GitHub OAuth | Ver.1.3-29 | 🔜 保留 | 未定 |
| GraphQL 対応 | Ver.1.3-29 | 🔜 保留 | 未定 |

## §5 Ver.2.0 開発対象

### §5.1 実施済み（Ver.2.0-40）

| 項目 | 内容 | ステータス |
|------|------|-----------|
| AP_VERSION 更新 | `'1.9.39'` → `'2.0.40'` | ✅ 完了 |
| DETAILED_DESIGN.md 全面改訂 | 旧エンジン名を Framework モジュール名に統一、Controller アーキテクチャ反映 | ✅ 完了 |
| ドキュメントバージョン更新 | README.md, Framework/README.md, features.md, ARCHITECTURE.md, SECURITY_POLICY.md, AdlairePlatform_Design.md | ✅ 完了 |
| MD5 パスワード自動移行コード除去 | index.php から MD5 検出・bcrypt 移行コードを削除 | ✅ 完了 |
| バージョンコメント整理 | index.php の Ver.1.x バージョン注釈を除去 | ✅ 完了 |
| CI ワークフロー廃止 | .github/workflows/ci.yml（PHP 構文チェック + ユニットテスト）を削除 | ✅ 完了 |
| AP.Bridge.php 廃止 | グローバル関数（json_read, h, host 等）を Framework クラスメソッドに直接置換し、ファイル削除 | ✅ 完了 |
| レガシーコメント整理 | Framework モジュール内の `Ver.1.x:` プレフィックス・`Bridge.php の〜相当` 等の旧コメントを除去 | ✅ 完了 |
| 単一パスワード認証廃止 | auth.json フォールバック除去、users.json に統一、`changeUserPassword()` 追加 | ✅ 完了 |
| WYSIWYG AEB EventBus 連携 | wysiwyg.js が AEB EventBus に `editor:save` / `editor:autosave` イベントを発行 | ✅ 完了 |

### §5.2 破壊的変更

| 変更 | 影響 | ステータス |
|------|------|-----------|
| MD5 パスワード非サポート | MD5 ハッシュの自動移行を廃止。Argon2id/bcrypt のみ | ✅ 実施済み |
| AP.Bridge.php 廃止 | グローバル関数（json_read, h, host 等）と JsonCache クラスを削除 | ✅ 実施済み |
| 単一パスワード認証廃止 | auth.json を使用する単一パスワード方式を廃止。users.json のみサポート | ✅ 実施済み |
| PHP 8.3+ 必須 | 変更なし（Ver.1.8 から継続） | ✅ 確定 |

### §5.3 将来の開発候補（Ver.2.0.x 以降）

| 項目 | 優先度 | 備考 |
|------|--------|------|
| ~~AP.Bridge.php グローバル関数の廃止~~ | — | ~~✅ Ver.2.0-40 で実施済み~~ |
| ~~単一パスワード認証の廃止~~ | — | ~~✅ Ver.2.0-40 で実施済み~~ |
| WYSIWYG AEB 完全移行 | 中 | wysiwyg.js の全機能を AEB に移植（インラインツール、スラッシュコマンド等） |
| Framework Public 独立化 | 低 | Composer/npm パッケージ化 |
