## §1 本補遺の目的

本補遺は、`FRAMEWORK_RULEBOOK.md`（本則）を補完し、**Ver.2.0 メジャーアップデート**に固有の方針決定・保留事項・不採用判断を記録するものです。

本則に定める規則（エンジン駆動モデル、命名規則、削除禁止事項等）は引き続き有効であり、本補遺はそれらを上書きしません。

---

## §2 Ver.2.0 破壊的変更方針

Ver.2.0 はメジャーバージョンアップであり、**後方互換性のない変更**を含みます。

| 方針 | 内容 | ステータス |
|------|------|-----------|
| バージョニング | Ver.2.0-40（累積リビジョン・リセット禁止 — VERSIONING.md 準拠） | ✅ 確定 |
| 後方互換性 | 破壊的変更を許容する | ✅ 確定 |
| PHP 要件 | PHP 8.3+ 必須（変更なし） | ✅ 確定 |
| フレームワーク構成 | 7 フレームワーク・18 エンジン体制を維持 | ✅ 確定 |

## §3 Ver.2.0 で不採用とした機能

以下の機能は Ver.2.0 での実装を**見送り**ました。

### §3.1 OpenAPI 自動生成

| 項目 | 内容 |
|------|------|
| 機能 | Swagger/OpenAPI 形式の API ドキュメント自動生成 |
| 初出 | Ver.1.3-29 リリースノート「次期バージョンで検討予定」 |
| 判断 | **不採用** |
| 理由 | Ver.2.0 スコープ外。API 層の安定化を優先する |
| ステータス | ❌ 不採用（Ver.2.0） |

## §4 保留事項（将来バージョンで再検討）

以下の機能は Ver.1.3-29 リリースノートで「次期バージョンで検討予定」とされたものですが、Ver.2.0 では着手せず**将来バージョンで再検討**とします。

### §4.1 マルチ環境（dev / staging / production）

| 項目 | 内容 |
|------|------|
| 機能 | Git ブランチと環境（開発・ステージング・本番）の連動 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | AIS.Deployment との統合設計が必要。単独で導入するとインフラ複雑性が増大する |

### §4.2 GitHub OAuth

| 項目 | 内容 |
|------|------|
| 機能 | GitHub OAuth による外部認証連携 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | 外部依存ゼロ原則（FRAMEWORK_RULEBOOK §2.2）との整合性を要検討。OAuth ライブラリの自前実装 or 例外許可が必要 |

### §4.3 GraphQL 対応

| 項目 | 内容 |
|------|------|
| 機能 | REST API に加えて GraphQL エンドポイントを提供 |
| 初出 | Ver.1.3-29 リリースノート |
| 判断 | **保留** — 将来バージョンで再検討 |
| 再検討時期 | 未定 |
| 備考 | REST API（ACE.Api）との併存が前提。GraphQL パーサーの自前実装コストが高い |

### §4.4 保留事項サマリー

| 機能 | 初出 | Ver.2.0 判断 | 再検討時期 |
|------|------|-------------|-----------|
| OpenAPI 自動生成 | Ver.1.3-29 | ❌ 不採用 | — |
| マルチ環境 (dev/staging/prod) | Ver.1.3-29 | 🔜 保留 | 未定 |
| GitHub OAuth | Ver.1.3-29 | 🔜 保留 | 未定 |
| GraphQL 対応 | Ver.1.3-29 | 🔜 保留 | 未定 |
